# Dataset Format Reference

## How the Data Was Generated

The dataset is **fully synthetic** — generated by [data_generation/generate_fake_data.py](../data_generation/generate_fake_data.py) using only the Python standard library.

The script was run with:

```bash
python3 generate_fake_data.py --num-users 100 --num-sessions 5000 --days 60
```

**What the generator does:**

1. **Creates 100 fake engineers** — each with a random name, email, OS config, terminal type, Claude Code version, engineering practice, seniority level, and location. All drawn from weighted distributions matching real observed data.

2. **Simulates 5,000 coding sessions** — each session is assigned a random engineer and a random start time within the 60-day window (70% during business hours 9–17, 30% at other times).

3. **Generates realistic event sequences per session** — each session contains multiple turns. Per turn:
   - A `user_prompt` event is emitted
   - Claude makes 1–30 `api_request` calls (log-normal distribution)
   - Each API call may trigger 0–3 tool calls (`tool_decision` + `tool_result`)
   - ~1.2% chance of an `api_error` followed by a retry

4. **Wraps events into CloudWatch-style log batches** — events are sorted by timestamp then grouped into batches of 1–10, mimicking how real telemetry arrives from AWS CloudWatch → S3.

5. **Outputs two files:**
   - `output/telemetry_logs.jsonl` — one batch per line
   - `output/employees.csv` — the employee directory

All numeric distributions (token counts, costs, durations, tool weights) were derived from real Claude Code telemetry data. The random seed defaults to `42` for reproducibility.

## telemetry_logs.jsonl

Each line is a CloudWatch-style batch:

```json
{
  "messageType": "DATA_MESSAGE",
  "owner": "123456789012",
  "logGroup": "/claude-code/telemetry",
  "logStream": "otel-collector",
  "logEvents": [ ... ],
  "year": 2025,
  "month": 12,
  "day": 3
}
```

Each item in `logEvents` has:
```json
{
  "id": "...",
  "timestamp": 1748908560000,
  "message": "<JSON string — parse this>"
}
```

The parsed `message` is the actual event:

```json
{
  "body": "<event_type>",
  "attributes": { ... },
  "scope": { "name": "com.anthropic.claude_code.events", "version": "2.1.45" },
  "resource": { ... }
}
```

---

## Event Types

All events share these base fields inside `attributes`:

```
event.timestamp     — ISO 8601, e.g. "2025-12-03T00:06:00.000Z"
session.id          — UUID grouping all events in one coding session
user.email          — join key to employees.csv
user.id             — SHA-256 hash of email
user.account_uuid   — UUID
organization.id     — UUID
terminal.type       — vscode | pycharm | WarpTerminal | iTerm2 | ...
event.name          — short name (same as body suffix)
```

    "user.email": "blake.patel@example.com",
    "user.id": "4ba41b6cc8bf83a2096c4bcd7c735f5dc746824a6153f5e0f9b359f3ee50b7a7",
    "event.name": "user_prompt",
    "prompt": "<REDACTED>",

### 2. `claude_code.api_request`

Fired for every LLM call Claude Code makes.

```json
{
  "body": "claude_code.api_request",
  "attributes": {
    "event.timestamp": "2025-12-03T00:06:00.275Z",
    "organization.id": "1f09f7dc-3388-4d71-b938-21a4f5001029",
    "session.id": "871c6054-b6b9-4336-b5dc-4df4317bffe6",
    "terminal.type": "vscode",
    "user.account_uuid": "03b2b865-a61c-49a6-9959-2f5c7e32cf70",
    "user.email": "blake.patel@example.com",
    "user.id": "4ba41b6cc8bf83a2096c4bcd7c735f5dc746824a6153f5e0f9b359f3ee50b7a7",
    "event.name": "api_request",
    "cache_creation_tokens": "0",
    "cache_read_tokens": "55024",
    "cost_usd": "0.005496101613304008",
    "duration_ms": "9078",
    "input_tokens": "0",
    "model": "claude-opus-4-5-20251101",
    "output_tokens": "939"
  }
}
```

**Unique fields:**
- `model` — e.g. `claude-haiku-4-5-20251001`, `claude-opus-4-6`, `claude-sonnet-4-6`
- `input_tokens` — fresh prompt tokens
- `output_tokens` — tokens generated
- `cache_read_tokens` — tokens served from prompt cache
- `cache_creation_tokens` — tokens written to prompt cache
- `cost_usd` — dollar cost of this single call
- `duration_ms` — end-to-end latency

### 3. `claude_code.tool_decision`

Fired when Claude proposes a tool call — before execution.

```json
{
  "body": "claude_code.tool_decision",
  "attributes": {
    "event.timestamp": "2025-12-03T00:06:09.353Z",
    "organization.id": "1f09f7dc-3388-4d71-b938-21a4f5001029",
    "session.id": "871c6054-b6b9-4336-b5dc-4df4317bffe6",
    "terminal.type": "vscode",
    "user.account_uuid": "03b2b865-a61c-49a6-9959-2f5c7e32cf70",
    "user.email": "blake.patel@example.com",
    "user.id": "4ba41b6cc8bf83a2096c4bcd7c735f5dc746824a6153f5e0f9b359f3ee50b7a7",
    "event.name": "tool_decision",
    "decision": "reject",
    "source": "user_reject",
    "tool_name": "Edit"

```

**Unique fields:**
- `tool_name` — Read | Bash | Edit | Grep | Glob | Write | TodoWrite | Task | WebFetch | WebSearch | mcp_tool | ...
- `decision` — `"accept"` or `"reject"`
- `source` — `"config"` | `"user_temporary"` | `"user_permanent"` | `"user_reject"`

Fired after a tool executes (only when decision was `accept`).

```json
{
  "body": "claude_code.tool_result",
  "attributes": {
    "event.timestamp": "2025-12-03T00:08:14.321Z",
    "organization.id": "1f09f7dc-3388-4d71-b938-21a4f5001029",
    "session.id": "871c6054-b6b9-4336-b5dc-4df4317bffe6",
    "terminal.type": "vscode",
    "user.account_uuid": "03b2b865-a61c-49a6-9959-2f5c7e32cf70",
    "user.email": "blake.patel@example.com",
    "user.id": "4ba41b6cc8bf83a2096c4bcd7c735f5dc746824a6153f5e0f9b359f3ee50b7a7",
    "event.name": "tool_result",
    "decision_source": "config",
    "decision_type": "accept",
    "duration_ms": "61",
    "success": "true",
    "tool_name": "Read"
  }

```

**Unique fields:**
- `tool_name` — same tool as the preceding `tool_decision`
- `decision_type` — `"accept"` or `"reject"`
- `decision_source` — `"config"` or `"user_reject"`
- `success` — `"true"` or `"false"`
- `duration_ms` — tool execution time
- `tool_result_size_bytes` — (optional) output size in bytes

### 5. `claude_code.api_error`

Fired when an API call fails.

```json
{
  "body": "claude_code.api_error",
  "attributes": {
    "event.timestamp": "2025-12-03T04:44:16.425Z",
    "organization.id": "53f91f78-8a0d-41ff-b950-c68c90d5ad7d",
    "session.id": "48cc03db-96b1-420e-8453-b1d40afe7148",
    "terminal.type": "vscode",
    "user.account_uuid": "ada35bbf-3bd8-447b-89ef-f3f7b1a37535",
    "user.email": "cameron.thomas@example.com",
    "user.id": "93e93dc80d51ed6082db67ef1adcca8777f000533d3622664c900219a13c05d0",
    "event.name": "api_error",
    "attempt": "2",
    "duration_ms": "943",

    "model": "claude-opus-4-5-20251101",
    "status_code": "401"

```

**Unique fields:**
- `model` — model that errored
- `error` — error message string
- `status_code` — `"400"` | `"401"` | `"429"` | `"500"` | `"undefined"`
- `attempt` — retry attempt number: 1, 2, or 3
- `duration_ms` — time elapsed before failure

## Session Flow

Events within a session follow this pattern, repeating per turn:

```
user_prompt
  └── api_request
        ├── [api_error → api_request retry]   (rare, ~1.2%)
        ├── tool_decision → tool_result
        ├── tool_decision → tool_result
        └── ...
  └── api_request  (next cycle)
        └── ...
[next user_prompt / turn]
```
